IDEA: Mimicking Backpropogation With Calculating Determinant

Theory

Assumptions: The determinant can only be found for square matrices, that is to say A is n by n.

Method: to calculate the determinant I utilized a special property of matrix determinants, which is that upper triangular matrices have a determinant equal to the product of their diagonal elements. To achieve this, I row reduced the matrix to an upper triangle and then multiplied the diagonal elements together. But when we row reduce we change the matrix and therefore change the determninant. However, we can keep track of the effects to the determinant by keeping track of the row operations we perform. For example, if we swap two rows, we multiply the determinant by -1. If we multiply a row by a scalar, we multiply the determinant by that scalar. And if we add a multiple of one row to another, we do not change the determinant. Thus, when we row reduce to an upper triangular matrix, we can multiply the diagonal elements together and then backproppogate through the previous steps to account for the matrix transformations.

Formula:

For each row:

  1. If the current pivot (when i = j / diagonal entry) is zero swap with the next nonzero row and change the determinant sign
  2. Add a scalar multiple of the current row to make the entries in the rows below zero

Then:

  1. multiply the diagonal and traceback to account for swap transformations

Note: One modification I made to the algorithm was that instead of backproppogating, I would just keep track of the changes to the determinant and pass it foward through the row operations.
